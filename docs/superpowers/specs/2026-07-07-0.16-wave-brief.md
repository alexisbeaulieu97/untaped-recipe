## Task A ‚Äî BUG: absolutize target/file paths before worker handoff

**Contract.** The engine resolves every target directory (and the per-step
`file` path derived from it) to an absolute path at the apply/hook-run
boundary, before any `HookRequest` is built. Hooks must never observe a path
whose meaning depends on the worker's cwd.

**Evidence.** `apply <recipe> app-alpha ...` (relative, valid from invoking
cwd): the worker resolved `target` against its own cwd (the pack project
dir); `target.rglob()` on the nonexistent path returned empty ‚Üí silent wrong
verdicts. Absolute paths passed.

**Test intent.** Regression: apply with *relative* CLI target paths + a
target-reading hook (fixture modeled on the exercise's `has_playbooks`) must
produce identical outcomes to the same invocation with absolute paths. Cover
`apply`, `apply --check`, and `hook run --target` with a relative path.

## Task B ‚Äî ruled: `add`/`check` uv.lock hookless exemption

**Contract.** `add` requires `uv.lock` under exactly the same condition
`check` does: only when the pack declares hooks. A hookless pack without a
lock installs cleanly; a hook-bearing pack without a lock still refuses with
the current error.

**Also (UX rider folded here):** `add` validates before printing the pack
summary ‚Äî error output leads, summary follows success only.

**Test intent.** Hookless lockless pack: `check` pass + `add --yes` succeeds.
Hooked lockless pack: both refuse. Message-order test: failed `add` emits no
pack summary.

## Task C ‚Äî `yaml_edit` `ensure` op

**Grammar.** New edit op alongside `set`/`merge`/`delete`:

```yaml
- op: ensure
  path: [collections]
  value: {name: acme.required}
  match: [name]          # optional; see semantics
```

**Semantics (ruled: match-keys + verbatim append, mapping dual).**

- Path resolves to a **list**: presence = any entry whose `match`-key values
  all equal `value`'s. Scalar `value` ‚Üí presence by equality, `match`
  forbidden (load error). Mapping `value` without `match` ‚Üí presence by
  whole-mapping equality. Absent ‚Üí append `value` **verbatim** (no style
  inference; authors choose the value shape for their fleet). In mixed
  str/mapping entry lists, string entries participate only in
  scalar-equality matching ‚Äî a mapping `value` never matches a string entry;
  document this explicitly.
- Path resolves to a **mapping**: `value` must be a mapping; each key absent
  from the target mapping is set (set-if-absent per key, shallow); existing
  keys are never modified. `match` is forbidden here (load error).
- Missing terminal container: created (like `set`), typed by `value` shape ‚Äî
  list ensure on a missing path creates the list with the one entry.
- Nothing to do ‚Üí the file's content is **byte-identical** (same no-op
  guarantee the exercise validated; no reformatting on untouched files).
- String values inside `value` use the same `{{ input }}` renderer and
  `unknown_tokens` policy as `set`/`merge` values.

**Design input from the exercise (must be honored in tests):** flow-style
lists stay flow-style with verbatim appends; comments/anchors/merge keys
survive; ruamel dump options are whatever `yaml_edit` already uses ‚Äî the op
adds no new formatting knobs.

**Test intent.** Golden-style fixtures derived from the exercise fleet's
`requirements.yml` variants: block list append (match by name), flow string
list scalar append, anchors/comments preservation, already-present ‚Üí
byte-identical, missing `collections:` key created, mapping set-if-absent,
mixed-entry matching, scalar+`match` load error, mapping-path+`match` load
error. Update the packaged built-in docs table.

## Task D ‚Äî outcome vocabulary: `skip` verdict + unified warn

**Ruled shape.** Validate verdict vocabulary becomes **pass / fail / skip**;
`helpers.warn()` becomes a **warning accumulator** callable from both hook
kinds; target status gains **`skipped`**.

**Contracts.**

- `helpers.skip(message)` ‚Äî new verdict return for validate hooks: the target
  is *not applicable*. Planning stops for that target; status = `skipped`;
  never counted as failure; `apply` exits 0 when every non-skipped target
  succeeds. Skipped targets appear in preview and summary with their own
  count ("8 targets: 5 applied, 2 unchanged, 1 skipped").
- `helpers.warn(message)` ‚Äî accumulator, callable any number of times from
  validate **and** transform hooks. Warnings attach to the target plan (the
  existing per-target warnings channel ‚Äî TargetPlan.warnings, the
  `recipe.outcome` `warnings` column, and sensitive-redaction already
  exist). The worker response schema gains a `warnings` list.
- Verdict **returns** slim to pass_/fail/skip. A legacy
  `{"status": "warn", ...}` verdict dict (or `helpers.warn(...)` used as a
  return value by old hooks) is accepted for this release and mapped to
  pass + accumulated warning; document as deprecated in hooks.md.
- `HOOK_API_VERSION` 0.9.0 ‚Üí 0.10.0; scaffold floors derive automatically
  (verify the derivation, don't hand-edit).
- **Pipe schema change (deliberate, sole-user):** `recipe.outcome.status`
  gains `skipped`; document in pipes.md with a one-line change note. No
  compatibility shim.
- Exit-code table in apply.md updated: skips are success; `--check` drift
  semantics unchanged.

**Test intent.** Skip verdict end-to-end (apply, --check, dry-run, pipe
record, exit code); warn accumulation from transform reaching outcome
warnings + preview; legacy warn-verdict mapping; boundary population: a run
where ALL targets skip (summary + exit 0 + no backup bundle); skip +
`--stdin` record flow. Regression: no-op runs still create no backup.
